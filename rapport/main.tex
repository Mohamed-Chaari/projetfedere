\chapter*{Résumé}
\addcontentsline{toc}{chapter}{Résumé}

\textbf{Français :}\\
Ce projet présente la création d'une plateforme Big Data pour collecter, stocker et analyser les données météorologiques de 221 villes en Tunisie. En utilisant des technologies modernes comme Kafka, Airflow et FastAPI, la plateforme permet de suivre la météo en temps réel et de détecter des anomalies climatiques à travers un tableau de bord interactif.

\vspace{0.8cm}
\textbf{English:}\\
This project presents the creation of a Big Data platform to collect, store, and analyze weather data from 221 cities in Tunisia. Using modern technologies like Kafka, Airflow, and FastAPI, the platform enables real-time weather tracking and climate anomaly detection through an interactive dashboard.

\chapter*{Conclusion et Perspectives}
\addcontentsline{toc}{chapter}{Conclusion et Perspectives}

\section*{Bilan du projet}
Ce projet nous a permis de concevoir et déployer une plateforme complète de suivi météorologique pour la Tunisie. Nous avons réussi à combiner des outils modernes du Big Data (Kafka, Supabase, Airflow) pour créer un système fiable et très réactif.

Aujourd'hui, l'application est fonctionnelle : elle collecte les données des 221 villes du pays, réalise des analyses statistiques de manière autonome, et affiche les résultats sur une interface web conviviale et intuitive.

\section*{Difficultés rencontrées}
Durant ce projet, nous avons dû surmonter plusieurs défis :
\begin{itemize}
  \item Apprendre à configurer et interconnecter plusieurs services Cloud.
  \item Gérer le grand volume de données tout en gardant un affichage web rapide.
\end{itemize}

\section*{Perspectives d'amélioration}
Pour aller plus loin à l'avenir, plusieurs pistes d'amélioration sont envisageables :
\begin{itemize}
  \item Intégrer des modèles d'Intelligence Artificielle pour améliorer les prévisions météo.
  \item Développer une application mobile (Android/iOS) pour les utilisateurs finaux.
  \item Ajouter un système de notifications par e-mail ou SMS en cas d'alerte météo majeure.
\end{itemize}

\chapter{Schéma SQL Complet}
\label{app:sql}

\begin{lstlisting}[language=SQL,caption={Extrait du schéma de la base de données Supabase}]
-- Table principale des mesures historiques
CREATE TABLE weather_historical (
    id            SERIAL PRIMARY KEY,
    date          DATE NOT NULL,
    city          VARCHAR(100) NOT NULL,
    governorate   VARCHAR(100),
    region        VARCHAR(50),
    temperature   REAL,
    humidity      REAL,
    precipitation REAL,
    wind_speed    REAL,
    ingested_at   TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(date, city)
);

-- Table des moyennes mensuelles (calculee par Airflow)
CREATE TABLE monthly_averages (
    id            SERIAL PRIMARY KEY,
    year          INTEGER NOT NULL,
    month         INTEGER NOT NULL,
    city          VARCHAR(100) NOT NULL,
    governorate   VARCHAR(100),
    region        VARCHAR(50),
    avg_temp      REAL,
    max_temp      REAL,
    min_temp      REAL,
    avg_humidity  REAL,
    avg_precip    REAL,
    avg_wind      REAL,
    record_count  INTEGER,
    computed_at   TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(year, month, city)
);

-- Table des correlations saisonnieres
CREATE TABLE correlations (
    id              SERIAL PRIMARY KEY,
    variable_a      VARCHAR(50) NOT NULL,
    variable_b      VARCHAR(50) NOT NULL,
    season          VARCHAR(20) NOT NULL,
    pearson_r       REAL,
    p_value         REAL,
    interpretation  VARCHAR(50),
    is_significant  BOOLEAN,
    computed_at     TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(variable_a, variable_b, season)
);
\end{lstlisting}

\chapter{Guide de Déploiement}
\label{app:deploy}

\section{Supabase}
\begin{enumerate}
  \item Créer un projet sur \url{https://supabase.com}
  \item Exécuter le schéma SQL (Annexe~\ref{app:sql}) dans l'éditeur SQL
  \item Récupérer le host du Session Pooler (Settings $\rightarrow$ Database)
\end{enumerate}

\section{Aiven Kafka}
\begin{enumerate}
  \item Créer un service Kafka sur \url{https://aiven.io}
  \item Télécharger les certificats SSL (ca.pem, service.cert, service.key)
  \item Créer les topics~: \texttt{weather-historical}, \texttt{weather-current}, \texttt{weather-forecast}
\end{enumerate}

\section{Astronomer Cloud (Airflow)}
\begin{enumerate}
  \item Installer Docker Desktop et le CLI Astro~: \texttt{brew install astro}
  \item Se connecter~: \texttt{astro login}
  \item Valider les DAGs~: \texttt{astro dev parse}
  \item Déployer~: \texttt{astro deploy}
  \item Configurer les variables d'environnement dans l'UI Astronomer
\end{enumerate}

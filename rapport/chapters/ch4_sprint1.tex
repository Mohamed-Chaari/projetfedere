\chapter{Sprint 1 « Fondations \& Flux Temps Réel »}

\section{Introduction}
L'objectif de ce premier sprint est de mettre en place l'architecture Cloud (Supabase, Kafka Aiven) et d'activer le flux de données en temps réel pour alimenter l'interface web.

\section{Objectif attendu et identification des tâches}

\subsection{Objectif attendu}
Déployer la base de données, configurer Kafka, charger l'historique météo et afficher les données en temps réel sur le tableau de bord.

\subsection{Sprint Backlog}
\begin{itemize}
    \item \textbf{US1 :} Configuration de Supabase et Aiven (Kafka).
    \item \textbf{US2 :} Développement du Producer et Consumer (pipeline).
    \item \textbf{US3 :} Création de l'API REST avec FastAPI.
    \item \textbf{US4 :} Authentification sécurisée des utilisateurs (JWT).
    \item \textbf{US5 :} Création du Dashboard interactif.
    \item \textbf{US6 :} Tests de validation de l'ingestion.
\end{itemize}

\subsection{Descriptions textuelles}
\textbf{Cas d'utilisation « S'authentifier »} : L'utilisateur entre son email et son mot de passe sur la page de connexion. Le système vérifie ces données dans la base Supabase et retourne un jeton (token) JWT en cas de succès pour sécuriser la session.

\textbf{Cas d'utilisation « Consulter Dashboard »} : L'utilisateur ouvre le dashboard. L'interface récupère les données via l'API FastAPI et affiche les informations météorologiques actuelles de manière dynamique.

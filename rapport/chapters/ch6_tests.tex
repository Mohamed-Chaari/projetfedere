\chapter{Tests Globaux et Validation Finale}

\section{Stratégie globale de test}
Afin de garantir le bon fonctionnement de notre plateforme, nous avons mis en place une stratégie de test couvrant l'ensemble du projet : du code source jusqu'à l'interface visuelle.

\section{Tests unitaires}
Les tests unitaires permettent de vérifier chaque petite partie du code indépendamment. Nous avons utilisé l'outil \texttt{pytest} en Python.
\begin{itemize}
    \item \textbf{Résultat} : 100\% de nos tests critiques (125 tests) ont réussi.
    \item Les calculs statistiques, la connexion à la base de données et les règles de validation des données fonctionnent tous correctement.
\end{itemize}

\section{Tests de performance}
Pour vérifier si notre application est fluide, nous avons mesuré plusieurs temps de réponse :
\begin{itemize}
    \item \textbf{Ingestion de données} : L'enregistrement des données temps réel de 221 villes prend moins de 10 secondes.
    \item \textbf{Affichage Web} : Le temps de chargement du tableau de bord est rapide, généralement inférieur à 2 secondes.
\end{itemize}

\section{Bilan du projet et validation}
Grâce à la méthode Scrum, l'équipe a pu terminer l'ensemble des fonctionnalités prévues (User Stories) dans les temps, respectant ainsi nos objectifs initiaux pour les deux sprints. Tous les tests sont concluants, la plateforme est donc validée et prête à être utilisée en conditions réelles.
